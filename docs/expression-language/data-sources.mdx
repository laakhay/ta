# Data Sources

Source attributes let expressions explicitly target dataset feeds.

## Source Types

- `ohlcv`
- `trades`
- `orderbook`
- `liquidation`

## Common Forms

```python
"BTCUSDT.price"                         # defaults to ohlcv source
"BTCUSDT.trades.volume"
"binance.BTCUSDT.orderbook.imbalance"
"BTC.USDT.perp.price"                  # base/quote + instrument style
"BTC.USDT.perp.trades.volume"
"BTCUSDT.h1.trades.volume"             # timeframe token in chain
```

## Valid Field Families

`ohlcv` fields:

- `price`, `close`, `open`, `high`, `low`, `volume`
- derived: `hlc3`, `ohlc4`, `hl2`, `typical_price`, `weighted_close`, `median_price`, `range`, `upper_wick`, `lower_wick`

`trades` fields:

- `volume`, `count`, `buy_volume`, `sell_volume`, `large_count`, `whale_count`, `avg_price`, `vwap`, `amount`

`orderbook` fields:

- `best_bid`, `best_ask`, `spread`, `spread_bps`, `mid_price`, `bid_depth`, `ask_depth`, `imbalance`, `pressure`

`liquidation` fields:

- `count`, `volume`, `value`, `long_count`, `short_count`, `long_value`, `short_value`, `large_count`, `large_value`

## Best Practices

- Prefer explicit source references in multi-source strategies.
- Keep symbol naming consistent with dataset keys.
- Validate expression against target exchange capabilities before execution.
