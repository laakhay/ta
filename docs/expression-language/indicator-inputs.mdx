# Indicator Inputs

Indicators can accept either default context fields or an explicit input expression.

## Default Input Pattern

```python
"sma(20)"          # uses close by default
"rsi(14)"          # uses close
```

## Explicit Input Expression

```python
"sma(BTCUSDT.price, period=20)"
"sma(BTCUSDT.trades.volume, period=20)"
"sma(BTCUSDT.high + BTCUSDT.low, period=10)"
```

The compiler places explicit input into `input_series` for runtime evaluation.

## Field Shorthand Pattern

For indicators that support a `field` param, shorthand forms like `rolling_mean(volume, period=20)` are normalized.

## Practical Pattern

Use explicit input expression when you want full control over source and preprocessing:

```python
"ema(BTCUSDT.orderbook.imbalance, period=21) > 0"
```

## Guardrail

If indicator parameter requires literals, nested expression values are rejected during parse.
