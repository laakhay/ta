# Registry & Catalog

Registry stores executable indicator handles and schemas.
Catalog converts this to richer metadata for APIs/UIs.

## Registry APIs

- `register(...)`
- `indicator(...)`
- `describe_indicator(name)`
- `describe_all()`
- `list_indicators()`
- `list_all_names()`
- `get_global_registry()`

## Catalog APIs

- `list_catalog()`
- `describe_indicator(name)` (catalog variant)
- parameter coercion helpers in `laakhay.ta.catalog`

## Common Use Cases

- power expression builder UIs from schema metadata,
- parameter validation and coercion,
- auto-generate docs/reference pages,
- classify indicators by category and output role.
